# Debugging Algorithms — Year 2 Computing

## Big Question
What happens when an algorithm goes wrong, and how do we fix it?

## What We're Learning
In this lesson you will learn what a **bug** is in computing, how to find errors (**debugging**), and how to fix and **test** your instructions.

## Key Computing Words
| Word | What it means |
|---|---|
| algorithm | a set of step-by-step instructions |
| bug | a mistake or error in an algorithm or program |
| debug | finding and fixing a bug |
| sequence | instructions in the correct order |
| test | trying out your algorithm to check it works |
| predict | guessing what will happen before you test |

## The Computing Explained

### What Is a Bug?
A bug is a **mistake** in your instructions. It might be:
- A step in the **wrong order**.
- A **missing** step.
- A **wrong** instruction.

### Example: Making a Sandwich
**Buggy algorithm:**
1. Spread the butter.
2. Get the bread.
3. Put on the filling.
4. Put the two slices together.

**Bug found!** You need the bread *before* you can spread the butter.

**Fixed algorithm:**
1. Get the bread.
2. Spread the butter.
3. Put on the filling.
4. Put the two slices together.

### How to Debug
1. **Read** through the algorithm carefully.
2. **Predict** what will happen at each step.
3. **Test** it — follow the steps exactly.
4. **Find** the step that goes wrong.
5. **Fix** it and test again.

### Testing Is Important
Always test your algorithm after fixing it. Sometimes fixing one bug can create a new one!

## Quick Quiz
1. What is a bug? → *A mistake in an algorithm or program*
2. What does "debug" mean? → *Finding and fixing a bug*
3. What should you do after fixing a bug? → *Test the algorithm again*
4. Name one type of bug. → *Wrong order, missing step, or wrong instruction*

## Computing in the Real World
Software developers debug code every day. Even apps and games have bugs that need fixing. Testing is a huge part of making technology work properly.

- [ ] I know what a bug is
