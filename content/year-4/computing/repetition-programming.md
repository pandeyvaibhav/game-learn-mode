# Repetition in Programming 🎯

## What We're Learning
We are learning about **loops** — a way to make a computer repeat instructions without writing them out again and again. You'll discover count-controlled loops, infinite loops, and why repetition makes programs shorter and smarter.

## Key Words

| Word | Meaning |
|------|---------|
| loop | A set of instructions that repeats |
| repeat | To do something again |
| count-controlled loop | A loop that runs a set number of times |
| infinite loop | A loop that never stops unless you tell it to |

## Let's Explore! 🔍

### Why Use Loops?
Imagine you want a character to take 100 steps forward. Without a loop you'd write "move forward" 100 times! With a loop you just say: **repeat 100 times: move forward**. Loops save time, reduce mistakes and make code easier to read.

### Count-Controlled Loops
A **count-controlled loop** repeats a fixed number of times. In Scratch you might use the **"repeat 10"** block. In Python you could write `for i in range(10):`. The computer counts each time through and stops when it reaches the target.

### Infinite Loops
An **infinite loop** runs forever — or until you stop the program. In Scratch the **"forever"** block is an infinite loop. These are useful for things that should always be happening, like checking if a key has been pressed. Be careful though — an accidental infinite loop can freeze your program!

## Did You Know? 🌟
The flashing cursor on your computer screen is controlled by an infinite loop — it blinks on and off continuously until you start typing!
